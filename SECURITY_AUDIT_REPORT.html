Критические проблемы (9)
CRIT-001
📁 .env.local.example
Строки 1-55
Файл содержит реальные токены и пароли
Файл содержит реальные TELEGRAM_BOT_TOKEN, DATABASE_URL с реальными данными. Это КРИТИЧЕСКАЯ уязвимость - любой с доступом к репозиторию может получить доступ к боту и БД.
Security
Рекомендация:
1. Удалить реальные значения из .env.local.example 2. Оставить только плейсхолдеры (например: TELEGRAM_BOT_TOKEN=<your_token_here>) 3. Добавить .env.local в .gitignore 4. Немедленно переротировать все токены и пароли 5. Создать новый DATABASE_URL с новым паролем
CRIT-002
📁 lib/db.ts
Строка 17
Слабая типизация параметров БД (any[])
Параметр params использует тип any[] вместо правильной типизации. Это позволяет передавать потенциально опасные данные без валидации, создавая риск SQL Injection атак.
Security
Рекомендация:
Заменить params?: any[] на params?: (string | number | boolean | null)[] export function query(text: string, params?: (string | number | boolean | null)[]) { return pool.query(text, params); }
CRIT-003
📁 lib/csrf.ts
Строка 20
Memory leak в CSRF store
CSRF store использует in-memory объект без лимитов. При длительной работе произойдёт утечка памяти - старые токены никогда не удаляются полностью.
Performance
Рекомендация:
1. Реализовать максимальный лимит на количество токенов (10000) 2. Добавить автоматическую очистку старых токенов 3. Использовать Redis вместо in-memory store в production const MAX_TOKENS = 10000; if (Object.keys(csrfStore).length > MAX_TOKENS) { cleanup(Date.now()); }
CRIT-004
📁 lib/rateLimit.ts
Строка 7
Race condition в rate limiting
Race condition в многопоточной среде. store может быть модифицирован одновременно несколькими запросами без синхронизации. Счётчик не атомарен.
Security
Решение:
Заменить on-memory store на Redis с атомарными операциями INCR и EXPIRE команды. В production используйте готовую библиотеку express-rate-limit или redis-rate-limit.
CRIT-005
📁 lib/bot/handlers.ts
Строка 15
Неправильное использование 'as any' при парсинге
Использование as any при парсинге startParam. Неправильная обработка - парсится строка вместо объекта. Может привести к неправильной обработке реферальных кодов.
Code Quality
Рекомендация:
const startParam = ctx.match?.[0] ? String(ctx.match[0]) : ''; if (startParam.startsWith('ref_')) { const refCode = startParam.replace('ref_', ''); if (!/^[A-Z0-9]{6}$/.test(refCode)) return; }
CRIT-006
📁 lib/useFetch.ts
Строка 36
Отсутствует timeout для fetch запроса
При зависании сервера браузер может бесконечно ждать ответа, блокируя UI приложения.
Performance
Решение:
Использовать AbortController с timeout (10 сек): const controller = new AbortController(); const timeoutId = setTimeout(() => controller.abort(), 10000); const response = await window.fetch(url, { ...mergedOptions, signal: controller.signal });
CRIT-007
📁 lib/migrate.ts
Строка 31
Миграции выполняются БЕЗ транзакций
Если миграция упадёт на середине, БД окажется в несогласованном состоянии. Это может привести к коррупции БД.
Security
Рекомендация:
Обернуть каждую миграцию в транзакцию BEGIN/COMMIT/ROLLBACK для гарантии консистентности данных.
CRIT-008
📁 lib/notifications.ts
Строки 34-47
Необработанные Promise rejections
Асинхронные функции без try-catch блоков. Unhandled promise rejection приводит к падению процесса.
Security
CRIT-009
📁 components/ProductImage.tsx
Строка 21
Использование <img> вместо Next.js <Image>
Нет оптимизации изображений, нет lazy loading, нет automatic format detection (WebP). Это замедляет приложение.
Performance
Решение:
Заменить на Next.js <Image> для автоматической оптимизации и lazy loading
