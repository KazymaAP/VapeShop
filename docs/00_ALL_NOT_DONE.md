### КРИТИЧЕСКАЯ ПРОБЛЕМА #1: ОТСУТСТВИЕ HMAC ВЕРИФИКАЦИИ INITDATA

**Файл:** `lib/auth.ts`, строки 61-98  
**Серьёзность:** 🔴 **КРИТИЧНО - УЯЗВИМОСТЬ БЕЗОПАСНОСТИ**  
**Статус:** ❌ НЕ НАЧАТО

**Проблема:**
Функция `parseInitData()` парсит данные от Telegram WebApp БЕЗ верификации HMAC-SHA256 подписи. Это означает, что любой хакер может подделать данные о пользователе, отправив:
```json
{ "initData": "user=%7B%22id%22%3A999999999%22%7D" }
```
И система примет его за администратора.

**Почему это критично:**
- Полная компрометация системы авторизации
- Хакер может выдать себя за любого пользователя (админ, менеджер и т.д.)
- Возможен доступ ко ВСЕМ защищённым API эндпоинтам
- Утечка/удаление всех данных в БД

**Решение:**
Реализовать верификацию подписи HMAC-SHA256:
```typescript
const crypto = require('crypto');
const bot_token = process.env.TELEGRAM_BOT_TOKEN;
const secret_key = crypto.createHmac('sha256', 'WebAppData').update(bot_token).digest();

function verifyInitData(initData: string): boolean {
  const params = new URLSearchParams(initData);
  const hash = params.get('hash');
  params.delete('hash');
  
  const data_check_string = Array.from(params.entries())
    .sort(([a], [b]) => a.localeCompare(b))
    .map(([k, v]) => `${k}=${v}`)
    .join('\n');
  
  const computed_hash = crypto.createHmac('sha256', secret_key)
    .update(data_check_string)
    .digest('hex');
  
  return computed_hash === hash;
}
```

**Ориентировочное время:** 2 часа (+ 1 час тестирования)

### КРИТИЧЕСКАЯ ПРОБЛЕМА #6: SQL INJECTION В ДИНАМИЧЕСКОМ UPDATE

**Файл:** `pages/api/admin/products.ts`, строки 27-45  
**Серьёзность:** 🔴 **КРИТИЧНО - ВОЗМОЖЕН ВЗЛОМ БД**  
**Статус:** ⚠️ ЧАСТИЧНО (параметризованные запросы, но логика рискованна)

**Проблема:**
```typescript
const fields: string[] = [];
// ...
await query(`UPDATE products SET ${fields.join(', ')} WHERE id = $${idx}`, values);
```

Если `fields` пустой массив, UPDATE упадёт. Если будет ошибка в логике - может быть injection.

**Решение:**
1. Проверить, что fields не пустой
2. Использовать RETURNING для получения старых и новых значений одним запросом:
```typescript
if (fields.length === 0) {
  return res.status(400).json({ error: 'No fields to update' });
}

const result = await query(
  `UPDATE products SET ${fields.join(', ')} WHERE id = $${idx} RETURNING *`,
  values
);
```

**Ориентировочное время:** 1 час

### КРИТИЧЕСКАЯ ПРОБЛЕМА #7: ОТСУТСТВУЕТ ТРАНЗАКЦИЯ ПРИ CSV ИМПОРТЕ

**Решение:**
Использовать BEGIN/COMMIT или ROLLBACK:
```typescript

}
```

### ВЫСОКИЙ ПРИОРИТЕТ #3: N+1 QUERY В /api/admin/orders

**Файл:** `pages/api/admin/orders.ts`, строки 18-39  
**Серьёзность:** 🟠 **ВЫСОКИЙ - ПРОИЗВОДИТЕЛЬНОСТЬ**  
**Статус:** ❌ ТРЕБУЕТСЯ ИСПРАВЛЕНИЕ

**Проблема:**
```typescript
const result = await query(`SELECT ... FROM orders ...`);

const orders = await Promise.all(result.rows.map(async (order) => {
  const historyRes = await query(`SELECT ... WHERE order_id = $1`, [order.id]);  // ← N запросов!
  const itemsRes = await query(`SELECT ... WHERE order_id = $1`, [order.id]);    // ← N запросов!
  return { ...order, history: historyRes.rows, items: itemsRes.rows };
}));
```

Если 1000 заказов - это 1 + 1000 + 1000 = 2001 запрос вместо 3!

**Решение:**
Использовать LEFT JOIN:
```typescript
const result = await query(`
  SELECT o.*, json_agg(oh.*) as history, json_agg(oi.*) as items
  FROM orders o
  LEFT JOIN order_history oh ON o.id = oh.order_id
  LEFT JOIN order_items oi ON o.id = oi.order_id
  GROUP BY o.id
`);
```

**Ориентировочное время:** 2 часа

```typescript
  text: string,

  retries = 3

  for (let i = 0; i < retries; i++) {
```

### ВЫСОКИЙ ПРИОРИТЕТ #14: ОТСУТСТВУЕТ RATE LIMITING

**Все критичные API**  
**Серьёзность:** 🟠 **ВЫСОКИЙ - БЕЗОПАСНОСТЬ**  
**Статус:** ❌ НЕ НАЧАТО

**Проблема:**
Хакер может отправить 1000000 запросов в секунду и перегрузить сервер.

**Решение:**
Использовать Middleware с Redis или памятью:
```typescript
const rateLimiter = (limit = 10, windowMs = 60000) => {
  const requests = new Map();
  
  return (req: NextApiRequest, res: NextApiResponse, next: any) => {
    const key = req.ip;
    const now = Date.now();
    const userRequests = requests.get(key) || [];
    
    const recentRequests = userRequests.filter((t: number) => t > now - windowMs);
    
    if (recentRequests.length >= limit) {
      return res.status(429).json({ error: 'Too many requests' });
    }
    
    recentRequests.push(now);
    requests.set(key, recentRequests);
    next();
  };
};
```
