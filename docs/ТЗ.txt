# Техническое задание (Telegram Mini App + Бот)  
**Версия 12.0 – полное, детальное описание со всеми разделами и примерами кода**

---

## 1. Цель и ЦА

**1.1. Цель проекта**  
Создать полноценный интернет-магазин для продажи вейпов, жидкостей, расходных материалов и никподов внутри Telegram. Решение состоит из двух частей:  
- **Telegram Mini App** – веб-приложение, которое открывается внутри Telegram. Здесь пользователь просматривает товары, добавляет в корзину, оформляет и оплачивает заказы. Администраторы и менеджеры управляют магазином через встроенную админ-панель.  
- **Telegram-бот** – ассистент, который отправляет уведомления о статусах заказов, позволяет быстро просмотреть историю заказов, получить реферальную ссылку, а администраторам – делать рассылки и импортировать товары.

**1.2. Целевая аудитория (ЦА)**  
- **Покупатели (B2C)** – физические лица с низким или средним чеком, которые ценят скорость оформления, удобный каталог и прозрачность статусов заказа.  
- **Администраторы** – владельцы магазина, управляющие ассортиментом, ценами, заказами и пользователями.  
- **Менеджеры** – сотрудники, обрабатывающие заказы (подтверждение, изменение статусов, коммуникация с покупателями).  
- **Курьеры** – сотрудники, доставляющие заказы, имеющие доступ только к необходимым данным.

**1.3. Бизнес-задачи**  
- Запустить работающий канал продаж в Telegram с минимальными затратами на разработку и хостинг (используются бесплатные или условно-бесплатные сервисы).  
- Обеспечить удобство для покупателей: быстрый поиск, фильтры, сохранение корзины, оплата Telegram Stars, визуальный трекинг заказов.  
- Дать администраторам мощные инструменты управления: импорт товаров из CSV (прайс-лист), активация товаров с выбором цены из трёх колонок, массовые операции, канбан-доска для заказов, управление ролями.  
- Автоматизировать коммуникацию через бота: уведомления о статусах, напоминания о брошенных корзинах, реферальная программа.

## 2. Технологический стек

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| Фронтенд Mini App | Next.js (React) + TypeScript + TailwindCSS | Серверный рендеринг для SEO, адаптивная вёрстка, строгая типизация |
| Бэкенд Mini App | Next.js API Routes (serverless) | Все API находятся в папке `pages/api`, не требуется отдельного сервера |
| База данных | Neon (PostgreSQL) | Бесплатный слой, совместимость с серверлесс, надёжное хранение |
| Хранение изображений | Supabase Storage | Бесплатный тариф, загрузка и хранение фото товаров |
| Платежи | Telegram Stars | Интеграция через Telegram Bot API, пользователи оплачивают звёздами |
| Уведомления | Telegram Bot API (grammY) | Бот отправляет сообщения о заказах, статусах, напоминания |
| Кэширование | Upstash Redis (опционально) | Кэш каталога, снижение нагрузки на БД |
| Деплой | Vercel | Автоматический деплой из GitHub, бесплатный тариф |

## 3. Дизайн и UI/UX

### 3.1. Общая концепция
Всё приложение выполнено в тёмной гамме с неоново-фиолетовыми акцентами. Фон – глубокий чёрный с оттенком `#0a0a0f`. Все карточки, модальные окна, панели имеют цвет `#111115` и тонкую границу `#2a2a33`. Акцентный цвет – `#c084fc` – используется для кнопок, иконок, ссылок, обводки активных элементов. При наведении на карточку товара она получает свечение (`box-shadow: 0 0 12px rgba(192,132,252,0.4)`) и слегка поднимается вверх (transform: translateY(-4px)). Все переходы плавные (0.2–0.3 секунды). Текст основного контента – `#ededee`, второстепенный – `#a1a1aa`. Шрифт – Inter (подключается из Google Fonts), для заголовков – 24–28px, для названий товаров – 16px, для цен – 20–24px полужирный, для мелких подписей – 12–14px.

### 3.2. Главный экран (каталог)
- **Верхняя панель (шапка)**: слева – логотип магазина (текст или SVG); справа – иконка корзины с бейджем количества товаров (белый кружок с числом). Под шапкой – строка поиска: поле ввода с иконкой лупы, внутри серый плейсхолдер «Поиск по названию или вкусу». Под строкой поиска – горизонтальный скролл с категориями (плиточные кнопки). Активная категория подсвечивается фиолетовым, неактивные – серым. Рядом – кнопка «Фильтры» (иконка слайдера). При нажатии выезжает модальное окно с фильтрами: слайдер цены, чекбоксы брендов, выпадающий список по крепости, по объёму, по количеству затяжек. Внизу модалки – кнопки «Сбросить» (серая) и «Применить» (фиолетовая).
- **Основная область**: сетка товаров – 2 колонки на узких телефонах, 3–4 на планшетах, 5–6 на десктопах. Каждая карточка – прямоугольник с закруглёнными углами 24px. В левом верхнем углу карточки могут быть бейджи: «Акция» (красный фон), «Хит» (оранжевый), «Новинка» (голубой). В центре карточки – изображение товара (если нет – иконка-заглушка). Ниже – название товара (одна строка, обрезается с троеточием). Под названием – характеристика (например, «30ML 20MG») мелким серым шрифтом. Следом – цена крупным фиолетовым шрифтом. Под ценой – остаток (зелёный, если >0, красный «нет в наличии»). Кнопка «В корзину» – фиолетовая, с иконкой корзины, на всю ширину карточки. При нажатии кнопка сжимается (scale 0.98), появляется ripple-эффект, из карточки вылетает маленькая фиолетовая иконка и летит к иконке корзины в правом верхнем углу (анимация). После анимации кнопка возвращается в исходное состояние, а бейдж корзины увеличивается на 1.
- **Нижняя навигация (tab bar)**: закреплена внизу экрана. Иконки: 🏠 Главная, 🔍 Поиск (если поиск вынесен отдельно, можно объединить), 🛒 Корзина, 👤 Профиль. Активная иконка подсвечена фиолетовым неоном, остальные – серые. При скролле вниз нижняя панель остаётся на месте, но слегка затемняется (backdrop-filter: blur).

### 3.3. Страница товара
- **Шапка**: стрелка «Назад» слева, название товара (обрезается), иконка корзины справа.
- **Фотогалерея**: основной блок – большая фотография (80% ширины экрана). Если фото несколько – внизу миниатюры (точки или уменьшенные картинки). При клике на фото открывается полноэкранный просмотр с возможностью увеличения жестом.
- **Блок характеристик**: ниже фотографии – название товара крупно, под ним характеристика (вкус, крепость, объём). Цена – крупно, фиолетовая. Если товар участвует в акции, рядом перечёркнутая старая цена серым.
- **Кнопки действий**: две кнопки – «В корзину» (фиолетовая) и «Купить в 1 клик» (светло-серая с фиолетовой обводкой). При нажатии «Купить в 1 клик» появляется модальное окно с полями «Имя» и «Телефон». После заполнения заказ создаётся и пользователь переходит к оплате.
- **Блок «С этим также покупают»**: горизонтальный скролл с карточками товаров (уменьшенный вариант). Каждая карточка содержит фото, название, цену и кнопку «+» для быстрого добавления.
- **Сравнение и избранное**: под кнопками – иконка «Сравнить» (два квадратика) и «В избранное» (сердечко). При нажатии они меняют цвет на фиолетовый, товар добавляется в соответствующий список.
- **Вопрос-ответ (FAQ) по товару**: раскрывающийся список с вопросами и ответами. Администратор может редактировать эти вопросы.
- **Отзывы**: список отзывов (аватар, имя, дата, текст). Если отзывов нет – сообщение «Пока нет отзывов. Будьте первым!». Внизу – кнопка «Написать отзыв» (активна только после получения заказа). При нажатии открывается модальное окно с текстовым полем и кнопкой «Отправить».

### 3.4. Корзина
- **Список товаров**: каждый элемент – горизонтальная карточка: фото, название, характеристика, цена, счётчик количества (кнопки «–» и «+», текущее число), иконка корзины для удаления. При удалении появляется всплывающее сообщение (тост) «Товар удалён» и элемент плавно исчезает.
- **Блок промокода**: поле ввода и кнопка «Применить». Если промокод валиден, отображается сумма скидки.
- **Выбор доставки**: две радиокнопки – «Самовывоз» и «Курьерская доставка». При выборе самовывоза появляется список точек (радиокнопки). При выборе курьера – поле «Адрес» (с автодополнением через API карт) и календарь выбора даты (только будущие даты).
- **Итоговая сумма**: отображается сумма товаров, скидка по промокоду, стоимость доставки (если применимо), итого. Кнопка «Оформить заказ» – крупная фиолетовая. При нажатии проверяются все поля, создаётся заказ и вызывается платёжный инвойс.

### 3.5. Личный кабинет (профиль)
- **Аватар**: иконка пользователя (по умолчанию) или фото из Telegram. Рядом – имя и Telegram username.
- **Вкладки**: «Мои заказы», «Избранное», «Мои адреса», «Реферальная программа», «Отзывы», «Настройки».
- **Вкладка «Мои заказы»**: список заказов, каждый – карточка: номер, дата, сумма, статус (цветной бейдж). При нажатии на карточку раскрываются детали: состав заказа (таблица с товарами), итоговая сумма, адрес доставки, 6-значный код (если статус `readyship`), кнопки «Повторить заказ» и «Отменить» (если статус new). Повтор заказа добавляет товары в корзину и перенаправляет на страницу корзины.
- **Вкладка «Избранное»**: сетка товаров (аналогично каталогу) с кнопкой «В корзину». У каждой карточки есть иконка сердечка, при повторном нажатии товар удаляется из избранного.
- **Вкладка «Мои адреса»**: список сохранённых адресов с возможностью добавить новый (кнопка «+»). Каждый адрес можно отредактировать или удалить.
- **Вкладка «Реферальная программа»**: отображается реферальная ссылка (кнопка «Скопировать»), количество приглашённых друзей, сумма начисленных бонусов (с указанием, что 1 бонус = 1 рубль), доступный баланс. Если баланс положительный, появляется кнопка «Использовать бонусы при оплате» (в корзине будет чекбокс).
- **Вкладка «Отзывы»**: список отзывов, которые пользователь оставил. Можно удалить свой отзыв.
- **Вкладка «Настройки»**: переключатель «Получать уведомления от бота» (вкл/выкл), переключатель «Тёмная тема» (авто/тёмная/светлая), кнопка «Выйти» (очистить локальные данные).

### 3.6. Админ-панель (интерфейс для администратора)
- **Боковое меню (сайдбар)**: на десктопе – фиксированная ширина 260px, фон `#0c0c12`. Пункты меню с иконками и текстом. Активный пункт выделен неоновой левой границей (3px) и полупрозрачным фоном. Пункты: «Дашборд», «Товары» (раскрывающийся список: Активные товары, Импорт CSV, Категории, Комплекты), «Заказы» (Все заказы, Канбан-доска), «Пользователи», «Маркетинг» (Промокоды, Реферальная программа), «Контент» (Страницы, Баннеры, FAQ), «Доставка» (Точки самовывоза, Настройки), «Настройки» (Уведомления, Общие), «Логи», «Резервное копирование». На мобильных – бургер-меню в левом верхнем углу, сайдбар выезжает слева.
- **Верхняя панель**: справа – бейдж с ролью (Администратор) и переключатель ролей (админ/менеджер/курьер). При смене роли интерфейс динамически меняется (скрываются недоступные разделы).
- **Дашборд**: карточки с метриками (выручка, заказы, пользователи, низкий остаток) – каждая карточка имеет тёмный фон и фиолетовую цифру. Ниже – график продаж (Recharts) с возможностью переключения по дням/неделям/месяцам. Справа – список последних 5 заказов и топ‑5 товаров. Кнопка «Обновить» обновляет данные без перезагрузки страницы.
- **Управление товарами (активные)**: таблица с пагинацией. Колонки: фото, название, артикул, цена, остаток, активен (чекбокс), действия (редактировать, удалить, копировать). Над таблицей – кнопки «Добавить товар» (открывает модалку с формой), «Массовое редактирование» (открывает модалку с выбором полей), «Импорт CSV» (ведёт на страницу импорта). В таблице есть чекбоксы для массового выбора. При нажатии «Редактировать» открывается модальное окно с полями: название, характеристика, бренд (выпадающий список), категория (древовидный), цена, остаток, загрузка фото (drag-and-drop). Сохранить – кнопка внизу.
- **Импорт CSV**: страница с областью перетаскивания файла. После загрузки файл парсится, показывается предварительный просмотр первых 10 строк с возможностью маппинга колонок. После подтверждения импорт сохраняется в `price_import`. На отдельной вкладке «Список импортированных товаров» – таблица с колонками: название, характеристика, три цены, дистрибьюторская, кнопка «Активировать». При активации открывается модалка: выбор цены (из трёх или дистрибьюторской), выбор категории, бренда, загрузка фото. После активации товар переносится в `products`.
- **Управление заказами**: таблица с фильтрами (статус, дата, город, сумма). Строки заказов – кликабельны. При клике открывается боковая панель с деталями: состав, данные покупателя, заметки менеджера (поле с текстом). В панели – кнопка «Изменить статус» (выпадающий список), «Сохранить заметку». Групповые операции – через чекбоксы: «Изменить статус», «Отправить уведомление». Кнопка «Экспорт CSV» экспортирует текущий фильтрованный список.
- **Канбан-доска**: доска с колонками по статусам. Карточка заказа содержит номер, имя покупателя, сумму, дату. Перетаскивание карточки в другую колонку меняет статус (после подтверждения). В карточке есть иконка для быстрого просмотра деталей.
- **Управление пользователями**: таблица с колонками: ID, имя, роль, дата регистрации, кол-во заказов, заблокирован (чекбокс). Действия: назначить роль (выпадающий список), заблокировать/разблокировать, просмотр активности (кнопка «Подробнее» – ведёт на страницу с заказами и историей).
- **Маркетинг**: страница промокодов – таблица с CRUD. При создании: код, тип скидки (процент/фикс), значение, даты начала и конца, лимит использований, мин. сумма заказа. Реферальная программа – переключатель «Включить», поле «Процент бонуса».
- **Контент**: страницы – список (о нас, контакты) с возможностью редактирования через WYSIWYG-редактор. Баннеры – список с возможностью загружать изображения, задавать ссылку, менять порядок (drag-and-drop). FAQ – список вопросов/ответов с кнопкой «Добавить», можно менять порядок.
- **Доставка**: точки самовывоза – таблица (название, адрес, активна). Кнопка «Добавить точку». Настройки доставки – поля: стоимость курьерской доставки, порог бесплатной доставки.
- **Настройки уведомлений**: чекбоксы для каждого типа события (новый заказ, смена статуса, готов к выдаче, брошенная корзина) с выбором целевой роли (админ, менеджер, курьер, покупатель).
- **Логи**: таблица с фильтрацией по пользователю, дате, действию. Экспорт CSV.
- **Резервное копирование**: кнопка «Создать бэкап» – скачать дамп базы данных в формате SQL.

### 3.7. Интерфейс менеджера и курьера
- **Менеджер**: сайдбар содержит только разделы «Заказы» (Все заказы, Канбан-доска) и «Клиенты» (только просмотр). В списке заказов есть кнопки «Изменить статус», «Добавить заметку», «Экспорт CSV». Нет доступа к товарам, пользователям, настройкам.
- **Курьер**: сайдбар – «Заказы на доставку» (статусы readyship и shipped), «История доставок» (выполненные). В списке заказов отображаются только те, что готовы к выдаче или в доставке. У каждого заказа есть кнопка «Взять в работу» (меняет статус на shipped) и «Выполнен» (открывает модальное окно с полем для ввода 6‑значного кода). Также есть кнопки «Позвонить» (tel:) и «Написать» (tg://). История доставок – просто список выполненных заказов с датами.

## 4. Роли и права доступа

| Раздел / Действие | Администратор (admin) | Менеджер (manager) | Курьер (seller) | Покупатель (buyer) |
|-------------------|----------------------|--------------------|-----------------|-------------------|

## 5. Функциональные требования

### 5.1. Покупатель (buyer)

#### 5.1.1. Регистрация и вход
При первом открытии Mini App через Telegram передаётся объект `initDataUnsafe.user`. На бэкенде проверяется, есть ли пользователь с таким `telegram_id`. Если нет – создаётся запись в таблице `users`, генерируется уникальный реферальный код (например, из 6 символов). Если пользователь уже существует – обновляется поле `last_seen`. При входе по реферальной ссылке (параметр `startapp=ref_КОД` в URL Mini App) в `users.referred_by` сохраняется `telegram_id` реферера. Регистрация происходит автоматически, без действий со стороны пользователя.

#### 5.1.2. Просмотр каталога и фильтрация
- **Пагинация** реализована через кнопку «Загрузить ещё» (load more) для экономии трафика. При скролле вниз автоматически подгружаются следующие 20 товаров.
- **Поиск** работает с debounce 300 мс. Запрос отправляется после паузы ввода, чтобы не нагружать сервер. Поиск осуществляется по полям `name` и `specification` (регистронезависимо, частичное совпадение). Результаты обновляются без перезагрузки страницы.
- **Фильтры** открываются в модальном окне. Слайдер цены – двухползунковый (мин и макс). Чекбоксы брендов подгружаются из таблицы `brands`. Выпадающие списки крепости, объёма и затяжек формируются на основе атрибутов товаров (JSON). Применение фильтров отправляет GET-запрос с параметрами, ответ обновляет список товаров.
- **Сортировка** – выпадающий список в правом верхнем углу каталога: по умолчанию (по новизне), по цене (возрастание), по цене (убывание), по популярности (количество просмотров).

#### 5.1.3. Корзина
- **Сохранение корзины** происходит автоматически при каждом изменении (добавление, удаление, изменение количества). Запрос отправляется на сервер, обновляется запись в таблице `carts`. Корзина привязана к `telegram_id`, поэтому доступна с любого устройства.
- **Промокоды** проверяются на сервере. При вводе кода и нажатии «Применить» отправляется POST-запрос. Если промокод валиден (не истёк, не превышен лимит, соответствует сумме заказа), возвращается сумма скидки и обновляется итоговая цена. Промокод может быть процентным или фиксированным.
- **Выбор доставки** влияет на стоимость: если выбран самовывоз, доставка бесплатна. Если курьер – применяется стоимость, заданная в настройках. При достижении суммы заказа порога бесплатной доставки стоимость обнуляется.
- **Быстрый заказ** – модальное окно с полями «Имя» и «Телефон». После отправки создаётся заказ, пользователь перенаправляется на страницу оплаты (инвойс).

#### 5.1.4. Оформление заказа и оплата
- После нажатия «Оформить заказ» создаётся запись в таблице `orders` со статусом `pending` (ожидает оплаты). Заказ содержит состав (из корзины), выбранный способ доставки, адрес, дату, промокод.
- Сервер вызывает метод Telegram Bot API `sendInvoice` с параметрами: название товара (например, «Заказ №123»), описание (перечень товаров), цена (в звёздах), payload (идентификатор заказа). Пользователь получает сообщение от бота с кнопкой «Оплатить».
- После нажатия кнопки открывается встроенный интерфейс Telegram Stars. После успешной оплаты Telegram отправляет боту `pre_checkout_query` (бот отвечает `answerPreCheckoutQuery` с `ok`) и `successful_payment`. Бот обновляет статус заказа на `new` и уведомляет администратора.
- Если оплата не произошла в течение 30 минут, заказ автоматически переводится в статус `cancelled` (можно реализовать через cron или проверку при каждом запросе).

#### 5.1.5. Управление заказами в личном кабинете
- **История заказов** отображает последние 20 заказов с пагинацией. Каждый заказ можно раскрыть, увидеть детали. Статус отображается цветным бейджем: новый – синий, подтверждён – зелёный, готов к выдаче – оранжевый, в доставке – фиолетовый, выполнен – серый, отменён – красный.
- **6-значный код** показывается в деталях заказа, когда статус `readyship`. Код генерируется на сервере (случайное число от 100000 до 999999) и хранится в поле `code_6digit` с временем истечения (через 24 часа). Покупатель может показать этот код курьеру.
- **Повтор заказа** – при нажатии кнопки «Повторить» товары из заказа добавляются в корзину, пользователь перенаправляется на страницу корзины.

#### 5.1.6. Избранное и сравнение
- **Избранное** сохраняется в БД. Иконка сердечка на карточке товара и на странице товара. При нажатии сердечко становится фиолетовым, товар добавляется в таблицу `wishlist`. При повторном нажатии – удаляется.
- **Сравнение** – до 4 товаров. При нажатии кнопки «Сравнить» на карточке товара он добавляется в список сравнения (хранится в localStorage для неавторизованных, в БД для авторизованных). Страница сравнения показывает товары в виде таблицы: характеристики (название, цена, бренд, остаток), можно удалить каждый. Сравнение доступно и на десктопе, и на мобильных.

#### 5.1.7. Отзывы
- Пользователь может оставить отзыв только после того, как заказ со статусом `done` включает данный товар. Кнопка «Написать отзыв» появляется в деталях заказа и на странице товара (если пользователь купил товар). Отзыв – только текст, без оценки. Отзыв публикуется сразу, но администратор может его удалить. На странице товара отзывы сортируются по дате (новые сверху). Есть кнопка «Понравилось» (лайк) – увеличивает счётчик полезности.

#### 5.1.8. Реферальная программа
- У каждого пользователя есть уникальный реферальный код, сгенерированный при регистрации. Реферальная ссылка имеет вид `https://t.me/ваш_бот?start=ref_КОД`. При переходе по ссылке бот сохраняет `referred_by` у нового пользователя.
- Когда приглашённый пользователь совершает первый заказ (статус `done`), рефереру начисляются бонусы (процент от суммы заказа, заданный в админке). Бонусы зачисляются на счёт (`users.bonus_balance`). Пользователь может использовать бонусы при оплате (в корзине появляется чекбокс «Использовать бонусы»). Бонусы списываются, но не могут быть выведены реальными деньгами.

### 5.2. Администратор (admin)

#### 5.2.1. Дашборд и метрики
- **Виджеты** обновляются при загрузке страницы и по кнопке «Обновить». Выручка рассчитывается по сумме всех заказов со статусом `done` за выбранный период (по умолчанию – месяц). Количество заказов – количество заказов со статусом `new`, `confirmed`, `readyship`, `shipped`, `done` за период. Пользователи – общее количество зарегистрированных. Низкий остаток – количество товаров с `stock` < 5 (настраивается).
- **График продаж** – линейный график, по оси X – дни, по оси Y – сумма. Можно переключать на недели/месяцы. Данные берутся из таблицы `orders` с фильтром по статусу `done`.
- **Топ‑5 товаров** – список товаров, отсортированных по количеству продаж (из `order_items`). Для каждого товара показывается название, количество проданных единиц, сумма.
- **Брошенные корзины** – количество корзин, созданных более 2 часов назад и не перешедших в заказ. При клике на виджет открывается список с возможностью отправить напоминание (через бота).

#### 5.2.2. Управление товарами (CRUD)
- **Добавление товара** – форма с полями: название, характеристика, бренд (выпадающий список), категория (выпадающий список, подгружается из `categories`), цена, остаток, загрузка изображений (до 5 штук, можно перетаскивать). Также есть чекбоксы «Акция», «Хит», «Новинка». При сохранении товар попадает в `products` и сразу становится видимым.
- **Редактирование** – аналогичная форма, данные подгружаются из БД. При изменении цены или остатка можно добавить комментарий, который попадёт в лог.
- **Копирование товара** – создаёт новый товар с такими же данными, кроме названия (добавляется «(копия)»). Позволяет быстро создавать похожие товары.
- **Массовое редактирование** – после выбора нескольких товаров (через чекбоксы) открывается модалка: можно изменить цену (процент или фиксированное значение), категорию, бренд, удалить товары.

#### 5.2.3. Импорт CSV (работа с прайс-листом)
- **Загрузка файла** – администратор может перетащить CSV-файл в область или выбрать его через диалог. Принимаются файлы с разделителем запятая, кодировка UTF-8. После загрузки сервер парсит файл и сохраняет данные в `price_import`. Показывается сообщение: «Импортировано X товаров, Y ошибок (неверный формат)».
- **Маппинг колонок** – если структура CSV не соответствует ожидаемой, можно вручную сопоставить колонки с полями БД. Это делается в интерфейсе перед импортом.
- **Список импортированных товаров** – таблица с пагинацией. Для каждого товара отображаются название, характеристика, три цены, дистрибьюторская цена, кнопка «Активировать». Можно выбрать несколько товаров и активировать их пакетно.
- **Активация** – при нажатии «Активировать» открывается модальное окно: выбор итоговой цены (из трёх колонок или дистрибьюторской), выбор категории (древовидный список), выбор бренда (можно создать новый), загрузка изображений (необязательно). После подтверждения товар переносится в `products`. Если товар с таким названием и характеристикой уже существует – предлагается обновить остаток и цену.

#### 5.2.4. Управление заказами
- **Все заказы** – таблица с колонками: номер (первые 8 символов UUID), дата, покупатель (имя), сумма, статус, действия. Можно фильтровать по статусу, дате, сумме, городу (по адресу). Клик по строке открывает боковую панель с деталями: состав (таблица с товарами), данные покупателя (телефон, адрес), поле заметки менеджера (textarea). В панели есть выпадающий список смены статуса и кнопка «Сохранить заметку».
- **Канбан-доска** – визуальная доска с колонками для каждого статуса. Карточки заказа можно перетаскивать (drag-and-drop) из одной колонки в другую. При перетаскивании появляется запрос подтверждения. После подтверждения статус заказа меняется, отправляется уведомление покупателю (если включено).
- **Групповые операции** – выбор нескольких заказов через чекбоксы, затем выбор действия: изменить статус (все на один), отправить уведомление (шаблонное). Также можно экспортировать выбранные заказы в CSV.
- **Экспорт CSV** – формирует файл со всеми полями заказа (номер, дата, покупатель, телефон, адрес, состав, сумма, статус, заметки). Можно экспортировать текущий фильтрованный список.

#### 5.2.5. Управление пользователями
- **Список пользователей** – таблица с колонками: Telegram ID, имя, роль, дата регистрации, последний вход, количество заказов, заблокирован. Поиск по имени или ID.
- **Назначение ролей** – выпадающий список в строке пользователя (покупатель, менеджер, курьер). Изменение роли сразу сохраняется.
- **Блокировка** – чекбокс «Заблокирован». Заблокированный пользователь не может оформлять заказы (при попытке видит сообщение).
- **Просмотр активности** – при клике на пользователя открывается страница с его историей: все заказы, корзина, избранное, отзывы. Можно также войти под его учётной записью (impersonate) – для отладки.

#### 5.2.6. Маркетинг
- **Промокоды** – страница со списком существующих промокодов и кнопкой «Создать». Форма создания: код (обязательно), тип скидки, значение, дата начала, дата окончания, максимальное количество использований, минимальная сумма заказа. При сохранении проверяется уникальность кода.
- **Реферальная программа** – переключатель «Включить», поле «Процент бонуса» (например, 10). При отключении бонусы больше не начисляются, но уже начисленные остаются.

#### 5.2.7. Контент
- **Страницы** – список статических страниц (slug, заголовок). При редактировании открывается WYSIWYG-редактор (например, ReactQuill) для форматирования текста. Можно сохранить как HTML.
- **Баннеры** – список баннеров, каждый с изображением (загружается через Supabase), ссылкой, порядковым номером. Порядок можно менять перетаскиванием.
- **FAQ** – список вопросов/ответов, можно добавить новый, изменить порядок. При сохранении данные обновляются в БД.

#### 5.2.8. Доставка
- **Точки самовывоза** – таблица: название, адрес, активна. Кнопка «Добавить точку» открывает модальное окно с полями.
- **Настройки доставки** – поля: стоимость курьерской доставки (рубли), порог бесплатной доставки (рубли). При изменении сохраняются в таблицу `settings` (отдельная таблица или JSON).

#### 5.2.9. Настройки уведомлений
- **Список событий** – чекбоксы для каждого типа: «Новый заказ» (отправлять администраторам и менеджерам), «Смена статуса» (покупателю), «Заказ готов к выдаче» (курьерам), «Брошенная корзина» (покупателю). Рядом с каждым чекбоксом – выпадающий список для выбора целевой роли (если применимо). Настройки хранятся в таблице `notification_settings`.

#### 5.2.10. Логи действий
- **Таблица логов** – колонки: время, пользователь (Telegram ID, имя), действие, детали (JSON). Фильтры по пользователю, дате, действию. Экспорт CSV.

#### 5.2.11. Резервное копирование
- **Кнопка «Создать бэкап»** – вызывает API, которое выполняет `pg_dump` (через команду или через запрос к Neon API). Скачиваемый файл – SQL-дамп.

### 5.3. Менеджер (manager)
- **Дашборд** – виджеты: количество заказов в статусе `new` и `confirmed`, среднее время обработки. График поступления заказов за 7 дней.
- **Список заказов** – фильтры, канбан-доска, групповые операции, экспорт CSV. Можно добавлять заметки к заказу.
- **Клиенты** – просмотр списка покупателей (без редактирования), переход к истории заказов.

### 5.4. Курьер (seller)
- **Список заказов** – отображаются заказы со статусами `readyship` и `shipped`.  
- **Кнопки действий** – «Взять в работу» (меняет статус на `shipped`), «Выполнен» (открывает модальное окно с полем для ввода 6‑значного кода).  
- **Контактные данные** – телефон (кнопка «Позвонить»), Telegram (кнопка «Написать»).  
- **История доставок** – список выполненных заказов с датами.

### 5.5. Telegram-бот

#### 5.5.1. Команды и клавиатуры
- **/start** – регистрирует пользователя (если нет), отправляет приветствие и главное меню (Reply Keyboard). В приветствии упоминается, что можно открыть магазин, посмотреть заказы, получить реферальную ссылку.
- **/menu** – дублирует главное меню.
- **/orders** – показывает последние 5 заказов с инлайн-кнопками «Подробнее». При нажатии «Подробнее» отправляется сообщение с деталями заказа (состав, сумма, статус, код, кнопки «Повторить заказ», «Отменить заказ»). Кнопка «Повторить заказ» отправляет ссылку на Mini App с параметром `?startapp=repeat_order_<id>`.
- **/referral** – отправляет сообщение с реферальной ссылкой и статистикой (количество приглашённых, начисленные бонусы, доступный баланс).
- **/help** – отправляет текст помощи (контакты поддержки, инструкция по использованию бота).

#### 5.5.2. Уведомления
- Бот подписан на события через вызовы из админ-панели. При изменении статуса заказа вызывается функция `sendOrderStatusUpdate`, которая отправляет сообщение покупателю (если настройки разрешают). Сообщение содержит номер заказа, новый статус и комментарий (если есть).
- При создании нового заказа (статус `new`) отправляется уведомление администраторам и менеджерам (список из env или из БД) с краткой информацией: номер, сумма, покупатель.
- При статусе `readyship` генерируется 6‑значный код, отправляется покупателю и курьеру (если указан в `SELLER_TELEGRAM_IDS`).
- Напоминание о брошенной корзине – раз в сутки проверяются корзины, которые не были оформлены более 2 часов, и отправляется сообщение с ссылкой на корзину (через Mini App).

#### 5.5.3. Административные функции
- **/admin** – доступно только тем, чей ID в `ADMIN_TELEGRAM_IDS`. Отображает инлайн-меню с кнопками:
  - **Рассылка** – бот запрашивает текст, затем подтверждение, затем отправляет сообщение всем пользователям (получает список из БД). Отправка происходит с задержкой 1 секунда между сообщениями, чтобы не превысить лимиты.
  - **Статистика** – отправляет сообщение с кратким отчётом: количество пользователей, заказов сегодня/неделю/месяц, выручка.
  - **Импорт CSV** – бот ожидает файл. После получения парсит, сохраняет в `price_import`, возвращает результат.
  - **Активация товаров** – бот выводит первые 10 неактивированных товаров с инлайн-кнопками для выбора цены. После выбора цены и подтверждения товар активируется.
  - **Управление ролями** – бот запрашивает Telegram ID и роль (manager или seller), затем обновляет роль в БД.

#### 5.5.4. Обработка платежей
- **Создание инвойса** – при оформлении заказа в Mini App сервер вызывает `bot.api.sendInvoice`. В payload передаётся `order_id`. Пользователь получает сообщение с кнопкой оплаты.
- **Pre‑checkout query** – бот отвечает `answerPreCheckoutQuery` с `ok`.
- **Successful payment** – бот получает сообщение об успешной оплате, извлекает `order_id` из payload, обновляет статус заказа на `new` и отправляет уведомление администратору.

### 7.2. Пример кода: получение списка товаров с фильтрами
```ts
// pages/api/products.ts
import type { NextApiRequest, NextApiResponse } from 'next';
import { query } from '../../lib/db';

export default async function handler(req: NextApiRequest, res: NextApiResponse) {
  if (req.method !== 'GET') return res.status(405).json({ error: 'Method not allowed' });

  const { category, brand, minPrice, maxPrice, search, sort, page = 1, limit = 20 } = req.query;

  let sql = `SELECT * FROM products WHERE is_active = true`;
  const params: any[] = [];

  if (category) {
    sql += ` AND category_id = $${params.length + 1}`;
    params.push(category);
  }
  if (brand) {
    sql += ` AND brand_id = $${params.length + 1}`;
    params.push(brand);
  }
  if (minPrice) {
    sql += ` AND price >= $${params.length + 1}`;
    params.push(Number(minPrice));
  }
  if (maxPrice) {
    sql += ` AND price <= $${params.length + 1}`;
    params.push(Number(maxPrice));
  }
  if (search) {
    sql += ` AND (name ILIKE $${params.length + 1} OR specification ILIKE $${params.length + 1})`;
    params.push(`%${search}%`);
  }

  switch (sort) {
    case 'price_asc': sql += ' ORDER BY price ASC'; break;
    case 'price_desc': sql += ' ORDER BY price DESC'; break;
    case 'newest': sql += ' ORDER BY created_at DESC'; break;
    case 'popular': sql += ' ORDER BY views DESC'; break;
    default: sql += ' ORDER BY created_at DESC';
  }

  const offset = (Number(page) - 1) * Number(limit);
  sql += ` LIMIT $${params.length + 1} OFFSET $${params.length + 2}`;
  params.push(Number(limit), offset);

  const result = await query(sql, params);
  res.status(200).json(result.rows);
}
```

### 7.3. Пример кода: импорт CSV (admin)
```ts
// pages/api/admin/import.ts
import type { NextApiRequest, NextApiResponse } from 'next';
import formidable from 'formidable';
import fs from 'fs';
import { parseCsv } from '../../../lib/utils/parseCsv';
import { query } from '../../../lib/db';

export const config = { api: { bodyParser: false } };

export default async function handler(req: NextApiRequest, res: NextApiResponse) {
  if (req.method !== 'POST') return res.status(405).json({ error: 'Method not allowed' });

  const form = formidable({});
  const [fields, files] = await form.parse(req);
  const file = files.file?.[0];
  if (!file) return res.status(400).json({ error: 'No file uploaded' });

  const content = fs.readFileSync(file.filepath, 'utf-8');
  const rows = parseCsv(content);

  let imported = 0;
  for (const row of rows) {
    await query(
      `INSERT INTO price_import 
        (name, specification, stock, price_tier_1, price_tier_2, price_tier_3, distributor_price)
       VALUES ($1, $2, $3, $4, $5, $6, $7)`,
      [
        row['Наименование'],
        row['Характеристика'],
        parseInt(row['КОЛИЧЕСТВО'] || '0'),
        parseFloat(row['до 50.000 р']?.replace(/\s/g, '') || '0'),
        parseFloat(row['от 50.000 р']?.replace(/\s/g, '') || '0'),
        parseFloat(row['от 100.000 р']?.replace(/\s/g, '') || '0'),
        parseFloat(row['ДИСТР.ЦЕНА']?.replace(/\s/g, '') || '0')
      ]
    );
    imported++;
  }

  res.status(200).json({ message: `Импортировано ${imported} товаров` });
}
```

import { Bot, webhookCallback } from 'grammy';

const bot = new Bot(process.env.TELEGRAM_BOT_TOKEN!);

  }

    keyboard: [
      [{ text: '📦 Мои заказы' }, { text: '🔗 Реферальная ссылка' }],

  };
  await ctx.reply(`Привет, ${user.first_name}! 👋\n\nЯ бот магазина Vape Shop.`, { reply_markup: keyboard });

## 8. Требования к интерфейсу (дополнительные)

- **Адаптивность** – приложение должно корректно отображаться на всех устройствах: телефонах (320px–640px), планшетах (641px–1024px), десктопах (1025px+). На десктопе админ-панель использует боковое меню, на мобильных – бургер.
- **Тёмная / светлая тема** – по умолчанию тёмная, но можно переключаться в профиле. Также поддерживается автоматическая подстройка под системную тему через медиавыражения.
- **Доступность** – все интерактивные элементы имеют достаточный контраст, поддерживают навигацию с клавиатуры (tab, enter). Для слабовидящих добавлены aria-метки.
- **Анимации** – все переходы плавные (0.2–0.3s). Скелетоны загрузки имеют shimmer-эффект. Анимация добавления в корзину – летящая иконка.
- **Пустые состояния** – для корзины, избранного, истории заказов отображаются иллюстрированные сообщения с кнопкой перехода в каталог.
- **Валидация форм** – в реальном времени (например, телефон в формате +7XXXXXXXXXX, дата не ранее завтрашнего дня). При ошибках под полем появляется красная подсказка.

## 9. Безопасность и производительность

- **JWT** с httpOnly cookie для сессий администраторов/менеджеров/курьеров.  
- **Rate limiting** – ограничение количества запросов с одного IP или telegram_id (Upstash Redis).  
- **Валидация** всех входных данных (Zod).  
- **CSP заголовки** для предотвращения XSS.  
- **HTTPS** – Vercel предоставляет по умолчанию.  
- **Кэширование каталога** – через Redis (инвалидация при обновлении товаров).  
- **Маскировка номеров телефонов в логах** – показывать только последние 4 цифры.  
- **Хранение паролей** – не требуется, так как аутентификация через Telegram.  
- **Проверка прав** на каждом API-роуте через middleware, который смотрит на роль пользователя.

## 11. Приоритеты реализации (MVP vs v2)

### MVP (обязательно для запуска)
- Каталог с плиткой, поиском, фильтрами (категории, цена, бренды).
- Карточка товара с фото, характеристиками, кнопкой «В корзину».
- Корзина (сохранение в БД), промокоды, выбор доставки (самовывоз / курьер).
- Оформление заказа, оплата Telegram Stars (инвойс через бота).
- Личный кабинет: история заказов, трекинг статусов, 6-значный код.
- Админ-панель: управление товарами (CRUD), импорт CSV, активация товаров, управление заказами (таблица, канбан), управление пользователями (блокировка, роли).
- Бот: команды `/start`, `/orders`, `/referral`, `/help`, уведомления о статусах, рассылка для админа.
- Роли: admin, manager, seller, buyer.
- Реферальная программа (базовая).
- Дизайн «Неон Glow».

### v2 (следующая очередь)
- Автоматический парсинг характеристик (крепость, объём, затяжки) из поля `specification`.
- Расширенная аналитика (когорты, время суток, тепловая карта).
- Импорт изображений по URL из CSV.
- Автоматическое определение бренда и категории.
- Массовое создание промокодов.
- Impersonate (вход под пользователем).
- Service Worker для офлайн-доступа к каталогу.
- Вебхуки для внешних систем.
