### 2️⃣ STAGING ОКРУЖЕНИЕ (ПРОТЕСТИРОВАТЬ)

#### Тесты безопасности:
- [ ] Попытаться обратиться к `/api/admin/products` БЕЗ `X-Telegram-Id` → должно вернуть 401
- [ ] Пользователь 1 пытается удалить адрес пользователя 2 → должно вернуть 403
- [ ] Пользователь пытается добавить отзыв от имени другого → должно вернуть 403
- [ ] Rate limit: отправить 15 запросов за 1 минуту → должно вернуть 429 после 10-го

#### Тесты функциональности:
- [ ] Создать заказ через Telegram Stars
- [ ] Проверить, что платёж ходит в бота, а не возвращается из API

- [ ] Добавить новый адрес доставки
- [ ] Создать заказ с доставкой

#### Тесты БД:
- [ ] Проверить, что индексы добавлены: `\d+ index_name`
- [ ] Запустить длинный запрос и убедиться, что он быстрый
- [ ] Проверить, что SSL включён: `SELECT ssl FROM pg_stat_ssl;`

#### Тесты ошибок:
- [ ] Попробуйте вызвать несуществующий endpoint → должен вернуть 404 с правильным форматом
- [ ] Попробуйте отправить некорректные данные → должна быть валидация

### 5️⃣ DEPLOYMENT (ВЫПОЛНИТЬ НА PRODUCTION)

**Вариант A: Vercel (рекомендуется для Next.js)**
```bash
# Если связан с GitHub:
# Просто пушнуть в main branch, Vercel автоматически задеплоит

# Или руками:
vercel --prod
```

**Вариант B: Docker + VPS**
```bash
# 1. Собрать образ
docker build -t vapeshop:1.0.0 .

# 2. Запустить контейнер
docker run -d \
  --name vapeshop-prod \
  -p 3000:3000 \
  -e NODE_ENV=production \
  -e NEON_DATABASE_URL=postgresql://... \
  -e TELEGRAM_BOT_TOKEN=... \
  vapeshop:1.0.0

# 3. Настроить nginx как reverse proxy
```

**Вариант C: PM2**
```bash
# 1. Build
npm run build

# 2. Запустить через PM2
pm2 start "npm start" --name "vapeshop"

# 3. Сделать постоянным
pm2 startup
pm2 save
```

### 7️⃣ ЕСЛИ БУДУТ ПРОБЛЕМЫ

**Ошибка подключения БД:**
- Проверить `NEON_DATABASE_URL` в .env.production
- Убедиться, что SSL: `rejectUnauthorized=true` на production

**Ошибка Telegram Bot:**
- Проверить `TELEGRAM_BOT_TOKEN` и `TELEGRAM_BOT_ID`
- Убедиться, что оба верные (ID без `:token` части)

**Ошибка авторизации:**
- Проверить, что `X-Telegram-Id` передаётся в заголовках
- Убедиться, что `ADMIN_TELEGRAM_IDS` правильные

**Медленная работа:**
- Проверить, что индексы БД созданы
- Запустить миграцию 009: `db/migrations/009_add_indexes.sql`

**Rate limiting срабатывает неправильно:**
- Помните, что это memory-based implementation
- Для production используйте Redis вместо памяти
