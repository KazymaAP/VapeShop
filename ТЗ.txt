```markdown
# Telegram Mini App + Bot — Vape Shop (полное техническое задание с примерами кода и структурой)

## 1. Цель проекта
Разработать Telegram Mini App (интернет-магазин вейпов, жидкостей, расходников) и Telegram‑бота, который дополняет приложение: позволяет быстро просматривать заказы, получать уведомления, управлять реферальной программой, а также даёт администраторам инструменты для рассылок и импорта товаров.

## 2. Технологический стек
- **Фронтенд Mini App:** Next.js (React) + TypeScript + TailwindCSS
- **Бэкенд Mini App:** Next.js API Routes (serverless)
- **База данных:** Neon (PostgreSQL)
- **Хранение изображений:** Supabase Storage
- **Платежи:** Telegram Stars (через `Telegram.WebApp.openInvoice` и Bot API)
- **Уведомления:** Telegram Bot API
- **Кэширование:** Upstash Redis (опционально)
- **Деплой:** Vercel (Mini App и бот в одном проекте, webhook)

## 3. Дизайн — «Неон Glow» (утверждён)
- **Цвета:** фон `#0a0a0f`, карточки `#111115`, границы `#2a2a33`, акцент `#c084fc` с неоновым свечением.
- **Карточки товаров:** плитка, при наведении — неоновая обводка, лёгкое смещение вверх.
- **Анимации:** ripple на кнопках, летящая иконка в корзину, скелетоны загрузки.
- **Адаптив:** мобильный first, нижний tab bar для покупателя, боковое меню для админки.
- **Переключатель ролей в админке:** select в верхней панели (admin / manager / seller).

## 4. Функциональные требования

### 4.1. Mini App (покупатель)
- **Каталог:** плитка товаров, пагинация, фильтры (категории, бренды, цена, крепость, объём, затяжки), сортировка, поиск.
- **Карточка товара:** фото, характеристики, цена, бейджи («Акция», «Хит», «Новинка»), остаток, блок «С этим также покупают», кнопка «Сравнить».
- **Корзина:** сохранение в БД, изменение количества, промокод, выбор доставки (самовывоз / курьер), дата, быстрый заказ.
- **Оплата:** Telegram Stars.
- **Личный кабинет:** история заказов (трекинг), повтор заказа, избранное, адреса, реферальная программа, отзывы.

### 4.2. Mini App (админка, менеджер, курьер)
- **Админ (admin):** полный CRUD товаров, импорт CSV (с тремя ценами), активация товаров, управление заказами (канбан‑доска, групповые операции, экспорт CSV), пользователями, ролями, промокодами, реферальной программой, контентом, настройками уведомлений, просмотр логов, резервное копирование.
- **Менеджер (manager):** просмотр всех заказов, изменение статусов, экспорт CSV, заметки, шаблоны ответов.
- **Курьер (seller):** видит заказы со статусами `readyship` и `shipped`, меняет статус на `shipped` и `done` (с вводом 6‑значного кода подтверждения).

### 4.3. Telegram-бот
- **Команды для покупателя:** `/start` (регистрация, главное меню), `/menu` (меню), `/orders` (история заказов), `/referral` (статистика рефералов), `/help`.
- **Кнопки:** Reply‑клавиатура «Мои заказы», «Реферальная ссылка», «Открыть магазин», «Помощь».
- **Инлайн‑клавиатуры:** детали заказа → «Повторить заказ» (открывает Mini App), «Отменить заказ», «Написать менеджеру».
- **Уведомления:** бот присылает покупателю смену статуса заказа, напоминание о брошенной корзине.
- **Реферальная программа:** генерация ссылки `https://t.me/ваш_бот?start=ref_КОД`, статистика.
- **Административные команды:** `/admin` (доступ только по `ADMIN_TELEGRAM_IDS`) → рассылка, статистика, импорт CSV, активация товаров, управление ролями.
- **Платежи:** после создания заказа бот отправляет инвойс; обрабатывает `pre_checkout_query` и `successful_payment`, обновляет статус заказа.

### 7.2. Пример обработчика импорта CSV (API‑роут `pages/api/admin/import.ts`)
```ts
import type { NextApiRequest, NextApiResponse } from 'next';
import formidable from 'formidable';
import fs from 'fs';
import { parseCsv } from '../../../lib/utils/parseCsv';
import { query } from '../../../lib/db';

export const config = { api: { bodyParser: false } };

export default async function handler(req: NextApiRequest, res: NextApiResponse) {
  if (req.method !== 'POST') return res.status(405).json({ error: 'Method not allowed' });

  const form = formidable({});
  const [fields, files] = await form.parse(req);
  const file = files.file?.[0];
  if (!file) return res.status(400).json({ error: 'No file uploaded' });

  const content = fs.readFileSync(file.filepath, 'utf-8');
  const rows = parseCsv(content); // массив объектов

  for (const row of rows) {
    await query(
      `INSERT INTO price_import 
        (name, specification, stock, price_tier_1, price_tier_2, price_tier_3, distributor_price)
       VALUES ($1, $2, $3, $4, $5, $6, $7)`,
      [
        row['Наименование'],
        row['Характеристика'],
        parseInt(row['КОЛИЧЕСТВО'] || '0'),
        parseFloat(row['до 50.000 р']?.replace(/\s/g, '') || '0'),
        parseFloat(row['от 50.000 р']?.replace(/\s/g, '') || '0'),
        parseFloat(row['от 100.000 р']?.replace(/\s/g, '') || '0'),
        parseFloat(row['ДИСТР.ЦЕНА']?.replace(/\s/g, '') || '0')
      ]
    );
  }

  res.status(200).json({ message: `Импортировано ${rows.length} товаров` });
}
```

### 7.3. Пример webhook для Telegram‑бота (`pages/api/bot.ts`)
```ts
import { Bot, webhookCallback } from 'grammy';
import { NextApiRequest, NextApiResponse } from 'next';

const bot = new Bot(process.env.TELEGRAM_BOT_TOKEN!);

// Импорт обработчиков (реальные функции в lib/bot/handlers)
import { handleStart, handleMenu, handleOrders, handleReferral, handleHelp } from '../../lib/bot/handlers';
import { handleAdminCallback } from '../../lib/bot/handlers';

bot.command('start', handleStart);
bot.command('menu', handleMenu);
bot.command('orders', handleOrders);
bot.command('referral', handleReferral);
bot.command('help', handleHelp);
bot.command('admin', async (ctx) => {
  const allowed = (process.env.ADMIN_TELEGRAM_IDS || '').split(',').map(Number);
  if (allowed.includes(ctx.from?.id)) {
    // показать админ-меню (inline keyboard)
    await ctx.reply('Админ-панель', { reply_markup: { inline_keyboard: [[...]] } });
  } else {
    await ctx.reply('Недостаточно прав');
  }
});

bot.on('callback_query:data', handleAdminCallback);

// Обработка платежей
bot.on('pre_checkout_query', async (ctx) => {
  await ctx.answerPreCheckoutQuery(true);
});
bot.on('message:successful_payment', async (ctx) => {
  const payment = ctx.message.successful_payment;
  // обновить статус заказа в БД
  await query('UPDATE orders SET status = $1, paid_at = NOW() WHERE id = $2', ['new', payment.invoice_payload]);
  await ctx.reply('Оплата прошла успешно! Спасибо за заказ.');
});

export default webhookCallback(bot, 'next-js');
```

## 8. Инструкция по развёртыванию

1. **Создать базу данных в Neon** и выполнить все SQL-скрипты из раздела 5.
2. **Создать Telegram бота** через BotFather, получить токен.
3. **Установить webhook** бота:  
   `https://api.telegram.org/bot<токен>/setWebhook?url=https://ваш-домен.vercel.app/api/bot`
4. **Настроить переменные окружения** в Vercel:
   - `TELEGRAM_BOT_TOKEN`
   - `NEON_DATABASE_URL`
   - `WEBAPP_URL` (https://ваш-домен.vercel.app)
   - `ADMIN_TELEGRAM_IDS` (ваш ID)
   - `MANAGER_TELEGRAM_IDS` (опционально)
   - `SELLER_TELEGRAM_IDS` (опционально)
   - `SUPABASE_URL`, `SUPABASE_ANON_KEY` (для хранения изображений)
5. **Деплой на Vercel** (подключить репозиторий).
6. **Проверить работу**:
   - Открыть Mini App через бота (кнопка «Открыть магазин»)
   - Зарегистрироваться, оформить заказ, оплатить Telegram Stars
   - Проверить уведомления бота
