1. ⚡ .env.local.example СОДЕРЖИТ РЕАЛЬНЫЕ ТОКЕНЫ И ПАРОЛИ
   ────────────────────────────────────────────────────────
   Файл: .env.local.example, строки 6, 13, 16
   Проблема: Рабочие credentials (TELEGRAM_BOT_TOKEN, DATABASE_URL) видны всем!
   
   Действие: НЕМЕДЛЕННО!
   ✓ Смените TELEGRAM_BOT_TOKEN через @BotFather
   ✓ Смените пароль БД Neon
   ✓ Замените реальные значения на placeholder'ы в .env.local.example
   ✓ Проверьте Git history - есть ли коммиты с credentials?

2. 🔓 SQL Injection через слабую типизацию
   ────────────────────────────────────────
   Файл: lib/db.ts, строка 17-18
   Проблема: export function query(text: string, params?: any[])
   Если передать объект вместо примитива, SQL парсер может сбиться
   
   Исправление:
   export function query<T = any>(text: string, params?: (string | number | boolean | null | Date)[])

3. 💾 Memory Leak в CSRF Store
   ────────────────────────────
   Файл: lib/csrf.ts, строки 20, 113-123
   Проблема: In-memory словарь csrfStore растет без ограничений
   На production (Vercel) память переполнится через неделю
   
   Решение:
   - Использовать Redis для CSRF tokens
   - Или реализовать LRU Cache с макс 10000 элементов
   - Или добавить очистку: if (Object.keys(csrfStore).length > 10000) csrfStore = {};

4. ⚙️ Race Condition в Rate Limiting
   ──────────────────────────────────
   Файл: lib/rateLimit.ts, строки 70-94
   Проблема: Операции с store[key] не синхронизированы
   Несколько serverless функций могут одновременно изменять счетчик
   
   Решение: Использовать Redis или добавить mutex/lock

5. 🔐 Опасный as any casting в auth
   ────────────────────────────────
   Файл: lib/auth.ts, строка 289
   Проблема: (req as any).telegramId = telegramId
   Теряется типизация, скрывают ошибки
   
   Исправление: Создать правильный интерфейс
   interface AuthenticatedRequest extends NextApiRequest {
     telegramId: number;
   }

6. ❌ Устарелый Next.js Link синтаксис
   ──────────────────────────────────
   Файлы: BannerSlider.tsx (строка 130), EmptyState.tsx (строка 90-94)
   Проблема: <Link href="..."><a>...</a></Link> - УСТАРЕЛО в Next.js 13+
   
   Исправление: <Link href="...">контент напрямую</Link>

7. 🖼️ Отсутствие оптимизации изображений
   ────────────────────────────────────
   Файл: components/ProductImage.tsx, строки 21-27
   Проблема: Использует <img> вместо next/image
   Результат: Нет lazy-loading, адаптивности, optimization
   
   Исправление: Использовать <Image> из next/image

8. 🎨 Динамические Tailwind классы НЕ РАБОТАЮТ
   ───────────────────────────────────────────
   Файл: components/ValidatedInput.tsx, строка 109-112
   Проблема: className={`text-${error.type === 'error' ? 'danger' : ...}`}
   Tailwind статический анализатор не видит динамические классы!
   
   Исправление: Использовать clsx()
   import clsx from 'clsx';
   className={clsx('text-sm', error.type === 'error' ? 'text-danger' : 'text-warning')}

9. 🏗️ Миграции БД без транзакций
   ────────────────────────────
   Файл: lib/migrate.ts, строки 30-36, 85-86
   Проблема: Если миграция частично выполнилась, БД останется в bad state
   
   Исправление: Обернуть в BEGIN/COMMIT/ROLLBACK транзакцию
